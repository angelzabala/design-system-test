    layout

    this section is apart from the rest because the main logic of the utilities is here. you can have as many breakpoints as you want following the same object structure from the biggest to the smalles breakpoint.

    breakpoints will use their value to create appropiate media queries together along with a grid system based in flexboxgrid and the number of columns selected for your design system.

    its important to note that container classes are not defined by default. having said that, a lot of things can be changed in any particular breakpoint, IE dispaly, position, alignment, columns that the element occupy, margin, padding, row and column gap among others

    Description

    these colors are going to be the colors that you use for your design system, the naming structure is very important because it will taylor you classes

    Map structure

    key Namekey Descriptionkey Typekey Value
    category

    a category is a map that can contain multiple maps of colors

    		String— none
    category.type

    a type is a map that contains the different values or shades that a color can have

    		String— none
    type.shade

    the type shades are the different tints or variants of a color in the design

    		String— none
    shade

    key:value with the shade and its respective value (can be expressed on any color notation that the user requires)

    		Map— none

    shadows

    $shadows: (
  level-1: 0px 0px 8px rgba(0, 0, 0, 0.12),
  level-2: 0px 0px 15px rgba(0, 0, 0, 0.24)
);

    Description

    shadows are important to keep a clean structure, don't have multiple shadows over all the documents your design system, the naming structure is very important because it will taylor you classes

    Example

    classes/variables

    --shadow-#{$level}: #{$value} => --shadow-level-1: 0px 0px 8px rgba(0, 0, 0, 0.12);
.shadow-#{$level} => .shadow-level-1;
